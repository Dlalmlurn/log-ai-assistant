# ADR-008: 生成器画像不等于系统行为 baseline

**Status:** accepted

## Decision

日志生成器中的用户画像、常用 IP、工作时间、部门角色和风险标签，只能作为数据生成先验。

系统行为 baseline 必须从正式入库后的历史日志和日级特征中统计得出。

新来源 IP、新设备和新来源地判断必须基于历史 baseline、日级特征或持久化 seen 表，不得只依赖检测进程内存。

## Context

为了生成真实感更强的数据，日志生成器通常需要预设用户画像。比如常用登录时间、常用网段、角色、部门和异常标签。

这些先验可以帮助生成有规律的数据，但如果系统直接读取这些配置作为 baseline，就会变成生成器配置回显，无法证明行为建模能力。

## Alternatives

| 方案 | 结论 | 原因 |
| --- | --- | --- |
| 直接把生成器画像作为 baseline | 不采用 | 不能证明系统从历史数据中学习行为。 |
| 无画像随机生成日志 | 不采用 | 数据缺少长期行为规律。 |
| 生成器提供先验，系统从入库历史中学习 baseline | 采用 | 既保证数据规律，也保证 baseline 数据驱动。 |

## Rationale

导师建议行为基线应基于历史数据，优先采用 T+1 建模。生成器画像只负责让数据具有可学习的模式，不能替代系统自己的统计过程。

新来源判断也需要稳定历史依据。只依赖进程内存会在重启或扩缩容后产生不稳定判断。

## Consequences

- 生成器配置不得直接作为 `UserBaseline` 输出。
- baseline 训练必须读取正式入库数据。
- baseline 必须记录训练窗口、样本量和置信度。
- 新来源判断需要持久化依据。
- 当样本不足时，需要 peer group、部门或全局 baseline 降级。
