# 日志分析 AI 助手 - 技术栈文档

## 项目简介

## 四人分工方案

**按数据流切分**，每人负责一段完整的子系统，学完后能给其他人讲清楚"数据怎么进来、怎么处理、怎么出去"。

| 组员 | 负责模块         | 核心技术                         | 难度 |
| ---- | ---------------- | -------------------------------- | ---- |
| A    | 日志采集与传输   | Filebeat + Kafka                 | 中   |
| B    | 流处理与日志解析 | Flink (PyFlink) + 正则/Logparser | 高   |
| C    | 存储与行为建模   | Elasticsearch + UEBA 建模        | 高   |
| D    | AI 研判与可视化  | LangChain + 通义千问 + Streamlit | 中   |

共同基础：Python 3.11 + Docker（四个人都要会基本操作）

------

### 为什么这样分

**A - 采集与传输**：项目的入口。搞清楚 Filebeat 怎么监听日志文件、Kafka Topic 怎么设计、消息怎么生产消费。工作量相对轻一些，可以同时负责 Docker Compose 的整体编排和环境维护，充当"基础设施"角色。

**B - 流处理与解析**：项目的核心难点之一。Flink 的流处理概念（窗口、watermark、Kafka connector）学习曲线陡，加上日志解析的正则编写，工作量大。建议团队里编程基础最扎实的人来。

**C - 存储与行为建模**：ES 的 DSL 查询语法是导师特别提到的难点，加上 UEBA 行为基线的统计逻辑（平均登录时段、API 调用频率、常用 IP 段），需要对数据分析有感觉。

**D - AI 与可视化**：LangChain 的链式调用 + Prompt 工程 + Streamlit 搭建看板。技术深度不算最高，但需要把前面三个人的成果串起来展示，对全局理解要求高。建议沟通能力强的人来，后期也负责每日简报的生成逻辑。

构建一个日志分析 AI 助手，采集企业日志（VPN/OA/API/系统等），通过结构化处理、用户行为建模（UEBA）、大模型智能研判，实现异常检测与安全态势可视化。

## 版本选择理由

### Python 3.11.9

- 需求要求 3.9+，3.11 在性能和生态成熟度上是当前最佳平衡点
- 3.12/3.13 部分第三方库（如 PyFlink）兼容性尚未完全覆盖

### Kafka 3.7.0 (KRaft 模式)

- KRaft 模式不再依赖 ZooKeeper，部署复杂度大幅降低
- 3.7.0 是 KRaft 正式 GA 后的稳定版本

### Flink 1.18.1

- 当前 LTS 级别稳定版，PyFlink 支持完善
- 原生 Kafka Connector 成熟

### Elasticsearch 8.13.4

- 支持全文检索 + DSL 查询，适合日志搜索场景
- 8.x 同时提供 `_sql` 接口，可用 SQL 语法查询作为过渡
- Elastic Stack（Filebeat + Logstash + ES）三者版本必须对齐

### LangChain 0.2.x

- 需求明确要求使用 LangChain 构建分析链
- 0.2 是重构后的稳定版，模块拆分清晰（langchain-core / langchain-community）

### 通义千问 (dashscope)

- 国内直连，无需翻墙，延迟低
- 成本远低于 OpenAI，适合学生项目
- 推荐模型：`qwen-plus`（性价比最优）或 `qwen-max`（效果最好）

### Streamlit 1.35.0

- 需求明确要求，Python 原生快速出原型
- 内置图表组件，适合安全看板场景

## 架构数据流

```
日志源 (VPN/OA/API/系统)
    │
    ▼
Filebeat 8.13.4 (采集)
    │
    ▼
Kafka 3.7.0 (消息队列/缓冲)
    │
    ├──▶ Logstash 8.13.4 (解析/结构化)
    │        │
    │        ▼
    │   Elasticsearch 8.13.4 (存储/检索)
    │
    └──▶ Flink 1.18.1 (实时流处理/清洗)
              │
              ▼
         行为基线建模 (UEBA)
              │
              ▼
         LangChain + 通义千问 (AI 研判)
              │
              ▼
         Streamlit (可视化看板 + 每日简报)
```

## 快速启动

### 1. 启动中间件

```bash
docker compose up -d
```

验证服务状态：

```bash
# Kafka
docker exec kafka /opt/kafka/bin/kafka-topics.sh --bootstrap-server localhost:9092 --list

# Elasticsearch
curl http://localhost:9200/_cluster/health?pretty

# Flink Dashboard
# 浏览器访问 http://localhost:8081
```

### 2. 安装 Python 依赖

```bash
python3.11 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 3. 配置通义千问 API Key

```bash
export DASHSCOPE_API_KEY="your-api-key-here"
```

申请地址：https://dashscope.console.aliyun.com/

## Elastic Stack 注意事项

- Filebeat / Logstash / Elasticsearch 三者版本号必须一致（均为 8.13.4）
- ES 8.x 默认开启安全认证，开发阶段可关闭：
  ```yaml
  # elasticsearch.yml 或 docker-compose environment
  xpack.security.enabled: false
  ```
- ES 查询主要使用 DSL，但也支持 `_sql` 接口：
  ```bash
  # DSL 查询示例
  curl -X GET "localhost:9200/vpn-logs/_search" -H 'Content-Type: application/json' -d'
  {
    "query": { "match": { "username": "admin" } }
  }'
  
  # SQL 查询（过渡用）
  curl -X POST "localhost:9200/_sql" -H 'Content-Type: application/json' -d'
  {
    "query": "SELECT username, COUNT(*) FROM \"vpn-logs\" GROUP BY username"
  }'
  ```
